## 参加して感じたこと（所感）

### 全体的な印象

今年のWebX 2026を一言で表すなら、「ステーブルコイン元年の幕開けを体感した場」だった。これまで頻繁に耳にしたNFT・DAOといったキーワードはほぼ影を消し、セッションから登壇者の発言まで全体を貫いたのは「ステーブルコイン」の一色。会場全体に実業ベースの熱量があり、投機・夢物語ではなく、制度・実装・普及という具体的な議論が繰り広げられていたことが強く印象に残った。

### 印象に残ったテーマ

特に記憶に残ったのは、JPYCとJPYSCをめぐる議論だ。JPYCはアパホテルのインバウンド決済など着実に実装事例を積み上げているが、1回100万円の送金上限が富裕層や大口ユースケースへの拡大を阻んでいる。一方のJPYSCはその制限がなく信託型の信頼性を持つが、現状はSBI VCトレード内のクローズドローンチにとどまる。どちらが勝つかというゼロサム競争ではなく、制度上の役割を分担しながら市場全体を広げていくという姿勢が両者に共通していたのは好印象だった。

三メガバンクのセッションは、想像以上に本音が出た場だった。パブリックブロックチェーンへの全面展開より行内・業界内での活用を優先する現実路線が伝わりつつも、「AIの登場でオンチェーン金融が実用段階に入った」という認識は三メガに共通しており、変化のスピード感が加速している印象を受けた。ステーブルコインによる「預金流出」リスクへの危機感は三メガに共通しており、だからこそ両建て戦略で自分たちの経済圏に取り込もうとしている。脅威を認識した上で攻守一体で臨む姿勢は、守りであると同時に外銀との競争を見据えた積極的な布石としても読み取れた。

### 気づきと認識の変化

日本でのステーブルコイン普及には、キラーアプリか国の政策的な後押しが不可欠だという感覚を強くした。技術面でも制度面でも素地は着々と整いつつあるが、一般消費者が「使いたい」と思う場面が日常の中にまだ少ない。インバウンド決済（来日した外国人がそのままステーブルコインで支払える環境）はその突破口として現実的に感じられた。また、ブロックチェーンはあくまでバックエンドの革新であり、一般消費者が意識しないままに浸透していく──その世界が2030年前後に静かに訪れるというシナリオには、かなりリアリティを感じた。

### NFT・権利のトークン化について（私見）

ブロックチェーンの主戦場は完全に金融に移行していた。ステーブルコイン・RWA・決済インフラの話ばかりで、「NFT」という言葉はほとんど聞かなくなった。

ただ、NFTが使われる未来はこれから本格的に広がると考えている。それは金融ではなく、**権利の移転**の領域で。

チケット・トレーディングカード・施設の利用権・会員権・デジタル書籍（本・雑誌・漫画）——これらに共通するのは、**価値があるのに移転に摩擦がある**ことだ。デジタル書籍は買っても「読む権利」を得るだけで、読み終わっても売れないし貸せない。チケットは正規の譲渡手段が限られ、ゴルフや会員権はいまだに紙や印鑑が前提のものも多い。

物理では当たり前だった「持つ・貸す・売る・譲る」が、デジタルになった途端に失われた権利は意外と多い。

ブロックチェーンが解決するのはデータを管理することではなく、「誰がその権利を持っているか」を改ざんできない形で共有し、ルール付きで移転できるようにすること。金融のRWAが先行するのは自然な流れで、決済・本人確認のインフラが整えば、その上で非金融の権利もトークン化されていく。

**NFTは流行らなかったのではなく、インフラになって名前が消える。** そんな未来を、願望も込めて想像している。

### エンジニア目線のトピック

岡部氏が着用していた「JPYC チョットワカル」Tシャツが印象的だった。日本のエンジニアコミュニティに古くから伝わる「Linuxチョットワカル」Tシャツへのオマージュで、「完全に理解した（入門直後の万能感）→ なにもわからない（複雑さに直面した絶望）→ チョットワカル（真の熟練者の謙虚な到達点）」というダニング＝クルーガー的な三段階は、ブロックチェーンの文脈に当てはめても妙な説得力がある。円ステーブルコインの最前線に立つ当事者がこういった遊び心を持っているのが、このコミュニティらしさを感じさせた。

### 渡辺創太氏のセッションを聞いて

4〜5年前からパラチェーンオークションを通じて追い続けてきた渡辺創太氏のセッションは、やはり格別だった。語り口は変わらず今時の若者風で親しみやすく、それでいて視座の広さと将来を見据えた思考の深さはまったく変わっていない。円ステーブルコイン×AI×トークン化資産という軸で日本のWeb3産業を引っ張ろうとしている姿に、改めて注目人物であることを確信した。「行ってよかった」と素直に思えるセッションだった。

### セッション 1：円ステーブルコイン元年をどう見る — JPYC×JPYSC、それぞれの答え

| 項目 | 内容 |
|------|------|
| ステージ | Binanceステージ |
| 時間 | 9:05〜9:35 |
| 登壇者 | 渡辺創太（Startale Group CEO）、岡部典孝（JPYC inc. CEO）、近藤智彦（SBI VC Trade 代表取締役社長）、各務貴仁（株式会社CoinPost 代表取締役社長） |

#### 主要ポイント

**制度の整理：JPYCとJPYSCの違い**
- **JPYC**：第1号電子決済手段。資金移動業ライセンスで発行。裏付け101%義務。加盟店契約なしで広く利用可能（デジタル現金的な性質）
- **JPYSC**：第3号電子決済手段。SBI新生信託銀行が発行する信託型。現状はSBI VCトレード口座内でのクローズドローンチ

**市場規模と目標**
- 現状：日本の円建てステーブルコインは約130億円規模（米ドル建ては50兆円超）
- 2027年目標：数千億〜1兆円規模へ拡大
- 日本から世界市場の10〜20%を取りたいという成長意欲

**最大の制度ボトルネック：100万円制限**
- 1回100万円の購入上限が大口資金流入・レンディングサービス成立を阻害
- 法改正には2〜3年を要する見通し。並行して規制改革要望・迂回策を検討中

**今後の展開**
- JPYCはLINEとの協業で発行量が2週間で約2倍に増加
- 将来：国際送金・アセットマネジメント・株主優待・配当払いへの展開
- ステーブルコインをオンチェーン金融の「血液」として位置づける構想

**競争より協調**
- JPYCとJPYSCは競合より「市場全体の拡大」を優先する姿勢
- 将来的にJPYSCがパブリックチェーンに出た場合、ペア形成が面白いとの期待

### セッション 2：決済大手会社と描く、ステーブルコインによるリテール決済の再構築

| 項目 | 内容 |
|------|------|
| ステージ | Binanceステージ |
| 時間 | 12:50〜13:30 |
| 登壇者 | Christian Rau（Mastercard SVP Digital Assets & Blockchain）、Dr. Arda Akartuna、SB Seker、Pranav Rastogi |

#### 主要ポイント

**ステーブルコインの本質的価値**
- 「リスクフリーに近い性質」「プログラマビリティ」「24時間365日のリアルタイム利用」が決済への適性の源泉
- Mastercardは「既存決済の置き換え」でなく「現在の決済ネットワークの進化形」として位置づけ

**「水面下」の決済革新**
- 消費者の支払い体験（水面上）は大きく変わらないが、清算・処理基盤（水面下）が根本的に変わる
- 承認表示は即時でも、実際の資金移動に24〜36時間かかる現状をステーブルコインが改善

**アジア市場の特性**
- アジア新興市場のステーブルコイン採用率が世界最高
- 背景：高い送金コスト・銀行アクセスの制約・既存手段の不便さ
- 一方で各国の規制姿勢・資本流出懸念・ライセンス整備状況には大きな差

**コンプライアンス課題**
- 透明性の高さがあっても制裁回避・不正金融利用のリスクは存在
- 自動スクリーニング・継続的モニタリング・リスク格付け体制の整備が必須
- スケール対応するコンプライアンス運用が不可欠

**2029年メインストリーム化の指標**
- 採用率の数値ではなく「一般利用者が自然に使え、詐欺が抑制された状態」
- 「お母さんが自然に理解して実際の取引で使えること」が目安

#### 主要ポイント

**主要流出事件と教訓**

| 事件 | 原因 | 教訓 |
|------|------|------|
| Coincheck（2018） | マルウェア感染→秘密鍵窃取 | ホット/コールドウォレットの資産分配設計が最大損失を左右 |
| Ronin Bridge（2022） | マルチシグ突破 | 形式上の分散ではなく「真の独立性」が必要 |
| DMMビットコイン（2024） | 委託先端末から侵入・TX改ざん | 委託先を含む統制の整備が不可欠 |
| Bybit（2025） | 正常フローのまま署名対象を改ざん | 「何に署名しているか」署名対象データの保護が必要 |

**秘密鍵ライフサイクルに基づく4業務分類**
1. **生成**：暗号学的安全な乱数・生成環境の安全性・鍵生成セレモニー・証跡管理
2. **保管**：秘密鍵本体・キーシェア・バックアップ・復元情報を同等レベルで保護
3. **利用**：トランザクション情報と署名ロジックの両方を保護
4. **継続更新**：システム（ソースコード含む）と運用設計を不正変更から保護

**実務上の重要論点**
- 「何を守るか」を先に定義してから対策を設計
- 委託先を含む体制全体に統制を効かせ継続的に点検
- 事業継続性（障害・破損時の復旧体制）まで含めた設計

### セッション 4：銀行はトークン化・暗号資産とどう向き合うか — 3メガバンクの未来戦略

| 項目 | 内容 |
|------|------|
| ステージ | CRYLステージ |
| 時間 | 15:20〜16:10 |
| 登壇者 | 磯和啓雄（三井住友フィナンシャルグループ 執行役専務・グループCDIO）、上ノ山信宏（みずほフィナンシャルグループ 執行役常務・グループCDTO）、山本忠司（三菱UFJフィナンシャル・グループ 執行役専務・グループCDTO）、関口慶太（日本経済新聞社 編集委員兼NIKKEI Financial副編集長、MC） |

#### 主要ポイント

**ステーブルコイン vs トークン化預金：両建て戦略**
- 三メガ共通の認識：「どちらか一方に賭けるのではなく、両方を用意する」
- 利用者は違いを意識せず使うため、銀行側で裏側を吸収するシームレスな設計が前提
- ステーブルコインが銀行経済圏の外で広がれば「預金流出」リスク→だから両方やる

**初期ユースケース：CMS（キャッシュマネジメントシステム）**
- 大企業・商社が抱える「海外拠点への資金滞留問題」を解決
- 現行CMSの課題：人手介在・カットオフタイム・時差による資金効率悪化
- 共通ステーブルコインで24時間・リアルタイムの資金集約が実現すれば大きな価値

**三メガ共同インフラの合理性**
- 個別発行（SMBC/みずほ/MUFG各自）→利用企業の管理負担増、採算が合わない
- 決済インフラは固定費が大きく一定の取扱量がないと成立しない
- 競争すべき領域と共同化すべき領域を分ける発想

**AML/KYCが最大の普及障壁**
- ステーブルコインでも送金のAML・KYC・反社確認義務は銀行振込と同じ
- オンチェーン化・高速化によりAML対応コストがむしろ増大する逆説
- ブロックチェーンの追跡性＋AI活用で共通KYCインフラ化の可能性

**AIがオンチェーン金融を実用段階に押し上げた**
- 「三年前では使えなかったが、今は使えるようになってきた」（三メガ共通認識）
- AIエージェントが資金を自動運用する世界が現実的になってきた
- オンチェーン金融は攻めの成長戦略であると同時に「外銀との競争における防衛戦略」

### セッション 5：Cryptoは、なぜ社会を変えきれないのか。〜異なる視点からみる、過去と今後の10年〜

| 項目 | 内容 |
|------|------|
| ステージ | Binanceステージ |
| 時間 | 15:55〜16:35 |
| 登壇者 | 渡辺創太（Startale Group CEO）、三浦和夫（シンプレクス株式会社 エグゼクティブプリンシパル）、近藤智彦（SBI VCトレード株式会社 代表取締役社長）、増田隆幸（NADA NEWS 編集長） |

#### 主要ポイント

**ブロックチェーンの本質的価値はバックエンド**
- ユーザーに見える変化より、バックエンドのコスト削減（最大1/1000）が本質
- 株式トークンになっても商品自体は変わらないが、提供側では運営コストが劇的に削減
- AIと異なり「見えにくいため社会に認識されるまで時間がかかる」

**普及の壁は技術でも制度でもない**
- 「日常生活や業務フローを実際に良くする具体的なサービスが不足」
- いきなり全面置き換えでなく、ステーブルコインの強みが活きる領域から事業化
- 理想UX：「ブロックチェーンを意識せず、気づかないうちに使っている状態」
- AWS/GCP移行のように利用者がバックエンドを意識せずコスト・利便性の恩恵だけ受ける形

**AIエージェントが爆発的普及のトリガー**
- AIエージェントは銀行口座でなくウォレットを基盤として活動するのが自然
- AIによるトランザクション量拡大がブロックチェーン利用を爆発的に広げる可能性
- 5〜6年程度で人間を超えるAI活動が見えてきた

**2030年の展望**
- クリプト・ブロックチェーン関連が金額ベースで全体の約10%（100〜200兆円規模）
- 既存金融の置き換わりが4年でかなり進む可能性
- ブロックチェーンは「特別なもの」でなく「当然の選択肢」になる

- **AIエージェント決済**：JPYCはどの企業のロボットにも使える汎用性が強み

### 主要テーマ横断の気づき
